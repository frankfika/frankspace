## 🚀 接下来要做什么

### 步骤 1：配置 Supabase 数据库（必须）

1. **创建 Supabase 账号**
   - 访问：https://supabase.com
   - 注册免费账号

2. **创建项目**
   - 点击 "New Project"
   - 项目名称：`frankfika-portfolio`
   - 选择离你最近的区域
   - 设置数据库密码（记住它！）

3. **获取凭证**
   - 进入项目后，点击左侧 "Settings" → "API"
   - 复制这两个值：
     - **Project URL**（例如：https://xxxxx.supabase.co）
     - **anon/public key**（一长串字符）

4. **更新 .env.local 文件**

   打开项目根目录的 `.env.local` 文件，替换这两行：

   ```env
   VITE_SUPABASE_URL=你的_Project_URL
   VITE_SUPABASE_ANON_KEY=你的_anon_key
   ```

5. **运行数据库迁移**
   - 在 Supabase 控制台，点击左侧 "SQL Editor"
   - 点击 "New query"
   - 打开项目中的 `supabase/migration.sql` 文件
   - 复制全部内容，粘贴到 SQL Editor
   - 点击 "Run"（或按 Cmd/Ctrl + Enter）
   - 看到 "Success. No rows returned" 就成功了！

6. **创建管理员账号**
   - 点击左侧 "Authentication" → "Users"
   - 点击 "Add user" → "Create new user"
   - 填写你的邮箱和密码
   - ✅ 勾选 "Auto Confirm User"
   - 点击 "Create user"

7. **重启开发服务器**

   在终端按 `Ctrl + C` 停止服务器，然后运行：
   ```bash
   npm run dev
   ```

## 🛠 常见问题

### Q: 登录后看不到内容？
A: 需要先在管理后台手动添加内容，或者从 `constants.ts` 导入初始数据到数据库。

### Q: 图片上传失败？
A: 需要在 Supabase Storage 中创建 `images` 存储桶：
   1. 点击左侧 "Storage"
   2. 点击 "New bucket"
   3. 名称：`images`
   4. ✅ 勾选 "Public bucket"
   5. 点击 "Create bucket"

### Q: 环境变量不生效？
A: 修改 `.env.local` 后需要重启开发服务器（Ctrl+C 然后 `npm run dev`）
